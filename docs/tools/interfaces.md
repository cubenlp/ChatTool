# 工具接口规范

本文档定义 ChatTool 工具模块对内对外接口形态，确保同一工具具备统一接入与复用能力。

## 目录结构

每个工具应位于 `src/chattool/tools/<name>/`，推荐结构如下：

```text
tools/<name>/
├── __init__.py
├── cli.py
└── mcp.py
```

- `__init__.py`：对外暴露稳定 Python API。
- `cli.py`：提供 `chattool <name>` 命令入口实现。
- `mcp.py`：提供 MCP 工具定义与调用入口。

## Python API 接口

- 所有工具核心能力由 `__init__.py` 导出，供其他模块直接复用。
- API 形态优先使用显式函数或类，不依赖 CLI 参数上下文。
- 参数应支持由上层注入，不在工具层写死环境值。

## CLI 接口

- CLI 负责参数解析与用户交互，不承载核心算法实现。
- 必要参数缺失时默认进入交互流程。
- 支持 `-i` 强制交互、`-I` 强制非交互并在缺参时报错。
- 敏感参数展示需默认脱敏。

## MCP 接口

- MCP 入口只做协议适配与参数转发，调用工具层 API 完成业务。
- 工具规格需可被 `catalog` 发现与筛选。
- 标签与可见性策略保持一致，避免同一能力出现多套命名。

## 依赖与导入

- 遵循最小 import 原则，将重量依赖放在函数内部按需加载。
- CLI/MCP 层禁止复制工具核心实现，统一调用 `tools/<name>/`。

## 新增工具落地清单

- 完成 `tools/<name>/` 目录与对外 API。
- 完成 CLI 接入与交互行为校验。
- 按需完成 MCP 接入并注册到目录清单。
- 补充 `docs/tools/<name>/index.md` 使用文档。
