# 任务驱动的技能与工具沉淀

本文档记录在执行任务过程中如何持续沉淀工具与技能，并保证项目结构一致。

## 基本流程

1. **先完成任务，再规范化**  
    先完成实际需求，再做文档、命名与结构整理。

2. **优先 CLI 交互**  
    能用 `chattool` CLI 完成的读取与导出，尽量使用 CLI，确保可复现与可审计。

3. **CLI 测试文档先行**  
    `cli-tests/` 下的测试必须先写 `.md`，再实现对应 `.py`，确保步骤与预期可审阅。

4. **能力归类**  
   - 通用、可复用：补充到 `src/`（工具层、CLI、MCP）。  
   - 任务特定：补充到 `skills/<name>/`。

5. **技能落地要求**  
   - 必须包含 `SKILL.md` 与 `SKILL.zh.md`。  
   - 分阶段输出约定：`0_` 表示原始或起点；`1_` 起为加工内容。  
   - 目录层级保持简洁，优先 `playground/<频道>/<主题>/`。

6. **任务后复盘**  
   - 做一次代码 review，统一命名、结构与文档。  
   - 更新导航（如 `mkdocs.yml`）与文档索引。

## 结果目标

- 每次任务都有产出：工具、技能或文档。  
- 项目结构不分叉，既能完成任务，也能积累长期能力。
